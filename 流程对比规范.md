# 流程对比规范

两个流程步骤相同、操作不同时（如博客的写作流程 vs 分类整理流程、创建 skill vs 完善 skill），用「顶部编号链 + 底部细则展开」结构处理。

## 如何选择

| 推荐方式 | 适用场景 | 原因 | 参考示例 |
|---------|---------|------|---------|
| 文本式 | SKILL.md / terminal / file:// | 不依赖渲染，纯文字即可阅读 | [博客 SKILL](record-to-hexo-blog/SKILL.md) |
| 流程图式 | 支持 mermaid 的环境（GitHub 等） | 可视化分支走向，适合多链对比 | [Skill SKILL](record-to-skill/SKILL.md) |

## 文本式

### 结构

```
## 路径A

→ A1: 步骤名 → A2: 步骤名 → A3: 步骤名 → A4: 步骤名 → A5: 步骤名

## 路径B

→ B1: 步骤名 → B2: 步骤名 → B3: 步骤名 → B4: 步骤名 → B5: 步骤名

## 细则

[A1/B1: 步骤名] → 说明（共用）
[A2: 步骤名] → 说明（A 特有）
[B2: 步骤名] → 说明（B 特有）
...
```

### 编号规则

- 步骤按顺序编号（1、2、3...），A/B 编号对齐
- 共用步骤：标为 `A1/B1`，细则写一条
- 不同步骤：分开标为 `A2`、`B2`，细则各写一条

### 细则格式

```
[编号: 步骤名] → 一句完整说明
```

- 编号用中括号包裹，如 `[A1/B1: 分类判断]`
- 步骤名和说明之间用 `→` 连接
- 说明尽量用完整句子，包含「做什么 + 为什么/注意什么」

## 流程图式

适用于支持 mermaid 渲染的环境（GitHub、部分文档站点）。

### 多链对齐

多条平行流程（如 A 写作 / B 归类整理 / C 完善）编号需按语义对齐：

| 行 | A 写作 | B 归类整理 | C 完善 |
|----|--------|-----------|-------|
| 3 | A3: 撰写+检查完善 | B3: 无 | C3: 撰写+检查完善 |
| 5 | A5: 用户确认 | B5: 用户确认 | C5: 用户确认 |

- 同一步骤语义的节点使用同一编号（如 A3/C3 都是"撰写+检查完善"）
- 不同步骤语义的节点用不同编号（A2/B2 操作不同）

### 占位规则

- **有语义的占位：** 标为可见的「无」（如 B3: 无），告诉读者此步无操作
- **纯布局占位：** 用空节点 + `hide` 样式隐藏，用于补齐跳数对齐行号

```css
/* 隐藏占位空节点 */
.hide { fill: none; stroke: none; }
```

### 样式规范

三条平行流程推荐配色：

| 流程 | 颜色 | CSS |
|------|------|-----|
| A 写作 | 绿 | `fill:#e8f5e9,stroke:#4caf50` |
| B 归类整理 | 蓝 | `fill:#e3f2fd,stroke:#2196f3` |
| C 完善 | 橙 | `fill:#fff3e0,stroke:#ff9800` |
| 共用节点 | 灰虚线 | `fill:#f5f5f5,stroke:#9e9e9e,stroke-dasharray: 5 5` |

## 设计原理

- 顶部链：一眼看清两个流程的步骤对比
- 细则展开：每步的完整说明不分散，不重复
- 编号对齐：序号相同 = 共用，序号不同 = 差异，无需额外标记
